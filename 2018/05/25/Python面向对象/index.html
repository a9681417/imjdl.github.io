说明

OOP Object Oriented Progranming 简称面向对象编程，是一种程序设计思想。
面向过程的程序设计把计算机程序视为一系列的命令集合，即一组函数的顺序执行，为了简化程序设计，面向过程把函数继续切分为子函数，即把大块函数通过切割分成小块函数来降低系统的复杂度。
面向对象的程序设计把计算机程序视为一组对象的集合，每个对象都可以接受其它对象发过来的消息，并处理这个消息，计算机程序的执行就是一系列消息在各对象之间的传递。
在Python中所有数据类型都可以视为对象，当然也可以自定义对象。自定义的对象就是class类的概念。面向对象的设计思想就是从自然界中来的，因为在自然界中，类(class)和实例(instance)概念是很自然的。class是一种抽象的概念，而实例是由class抽象出来的。
面向对象三大特点封装、继承和多态。当然抽象也是特点之一。

由于类起到模板的作用，因此，可以在创建类的实例的时候，把一些我们必须绑定的属性强制填写进去

在类中定义函数和普通函数相比，在类中定义的第一个参数是实例变量self（可能这里不太准确，以后会详细说明），并且，调用时，不用传递该参数。除此之外，类的方法和普通函数没有什么区别，你仍然可以用默认参数、可变参数、关键参数和命名关键字参数

封装

封装或者说包装、信息隐藏，是指利用抽象数据类型将数据和基于数据的操作封装在一起。系统的其它对象只能通过包裹在数据外面的已经授权的操作来与这个封装对象交流和交互。用户无需知道对象内部实现的细节，但是可以通过该对象对外提供的接口来访问。

好处：减少耦合，类内部的结构可以自由修改

访问限制

定义类时如果没有限制，类的所有自定义的属性和方法都为public可以被随意访问。
如果不想被随意访问就在属性或方法名前加两个下划线，这样就变为私有属性或方法了

当然我们也见过一个下划线开头的，这样的变量或方法外部也是可以访问的，意思为”请把我视为一个私有变量，不要随意访问”
即使是两个下划线其实也是可以访问的

“私有”属性存在两种不同的命名规则（单下划线和双下划线）。对于大部分情况，我们应该让非公有名称以单下划线开头，如果涉及到子类化处理，并且内部属性需要对子类隐藏时用双下划线。

继承

多态

一个类继承了另一个或多个类并且重写了父类里的方法就为多态

多态性（注意：多态与多态性是两种概念）

多态性是指具有不同功能的函数可以使用相同的函数名，这样就可以用一个函数名调用不同内容的函数。在面向对象方法中一般是这样表述多态性：向不同的对象发送同一条消息，不同的对象在接收时会产生不同的行为（即方法）。也就是说，每个对象可以用自己的方式去响应共同的消息。所谓消息，就是调用函数，不同的行为就是指不同的实现，即执行不同的函数。
