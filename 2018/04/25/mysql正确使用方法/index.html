    Mysql的正确使用方法

    elloit blogs
    2018-04-25
    Mysql

    0x01 说明

    平时做web开发避免不了与数据库打交道，但是我以前总是给出mysql root 用户去操作数据库后来发现这样做是不对的，我们应当根据需求创建相应的普通用户，让后赋予他相关数据库的所需的权限。为了避免以后忘记这里做一下记录。

    一般普通用户权限有：select,update,delete,insert;

    A)授予权限
    B)撤销权限

    撤销update权限
    revoke update on youdatabase.* from youname@'localhost';
    撤销所有权限
    revoke all on youdatabase.* from youname@'localhost';

    0x07导入导出

    A)导出数据库用mysqldump命令（注意mysql的安装路径，即此命令的路径）：
    1、导出数据和表结构：
mysqldump -u用户名 -p密码 数据库名 > 数据库名.sql
#/usr/local/mysql/bin/   mysqldump -uroot -p abc > abc.sql
敲回车后会提示输入密码
2、只导出表结构
mysqldump -u用户名 -p密码 -d 数据库名 > 数据库名.sql
#/usr/local/mysql/bin/   mysqldump -uroot -p -d abc > abc.sql

注：/usr/local/mysql/bin/  --->  mysql的data目录
    B)导入数据库
    1、首先建空数据库
`mysql>create database abc default character set utf8;`

2、导入数据库
方法一：
(1)选择数据库
mysql>use abc;
(2)设置数据库编码
mysql>set names utf8;
(3)导入数据（注意sql文件的路径）
mysql>source /home/abc/abc.sql;
方法二：
mysql -u用户名 -p密码 数据库名 < 数据库名.sql
#mysql -uabc_f -p abc < abc.sql

    0x08 忘记密码

    时间长了，可能会忘记mysql的密码，登录是总是失败。

    1. Linux 下找到 /etc/mysql/debian.cnf
2. cat 打开后可以看到里面存储的系统管理员的帐号和密码
3. 以管理员身份登录
4. `use mysql;update user set password=PASSWORD('新密码') where user='root';FLUSH PRIVILEGES;`
5. 如果mysql是5.7往后的，user表里的password字段变为了authentication_string，该一下就好了

    结尾

    以上操作需要在root用户下完成，root用户是用来管理用户操作的而不是直接用来操作数据库的
    以后一定要改掉这个坏习惯。。。
